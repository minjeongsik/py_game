현재 프로그램 구성 방향 요약

1. 게임 장르/목표
- 2D 크리처 수집형 RPG의 세로 슬라이스(vertical slice) 형태
- 핵심 목표는 "최소한의 완성된 플레이 루프"를 만드는 것
- 현재 루프는 타이틀 -> 월드 탐험 -> 야생 조우 -> 전투 -> 포획/도주/승리 -> 저장/이어하기

2. 현재 구현된 핵심 기능
- 타이틀 메뉴: New Game / Continue / Exit
- 월드 이동: 플레이어 이동, 카메라 추적, 충돌 처리
- 지역 전환: 포탈 기반으로 다른 맵으로 이동
- 랜덤 인카운터: 특정 타일에서 확률적으로 야생 크리처 등장
- 전투 시스템: Attack / Item / Capture / Run
- 크리처 포획: 파티가 차면 스토리지로 이동
- 인벤토리: 회복 아이템과 포획 아이템 사용 가능
- 저장/불러오기: 위치, 파티, 스토리지, 인벤토리 저장
- 디버그 오버레이: 상태, 좌표, 입력, 파티 수 등 표시

3. 구조적으로 보이는 방향성
- 데이터 중심 설계: 종족, 기술, 아이템, 지역 정보를 JSON으로 분리
- 상태 기반 구조: Title, WorldExploration, Battle, PauseMenu
- 화면/월드/전투를 한 프로젝트 안에서 빠르게 구현하는 프로토타입 성격
- 장기적으로는 Game1.cs에 몰린 책임을 모듈별로 분리할 여지가 큼

4. 현재 부족하거나 다음 단계로 보이는 부분
- 전투 밸런스와 상태이상, 속성 상성, AI 등은 아직 단순함
- UI가 기능 중심이며 시각적 완성도는 아직 초기 단계
- NPC, 퀘스트, 상점, 대화 시스템은 거의 없거나 미구현 상태
- 맵/콘텐츠 수가 적고 확장용 구조는 있으나 실제 볼륨은 적음
- Game1.cs가 너무 많은 역할을 갖고 있어 유지보수성이 떨어질 수 있음

5. ChatGPT에 물어보면 좋은 방향
- 현재 구조를 유지하면서 가장 작은 리팩터링 우선순위가 무엇인지
- Game1.cs를 어떤 기준으로 분리하면 안전한지
- 데이터 중심 구조를 유지하면서 콘텐츠 확장을 쉽게 하는 방법
- 전투/월드/세이브 시스템 중 다음으로 강화할 핵심 축이 무엇인지
- vertical slice 다음 단계에서 MVP를 어떻게 정의하면 좋은지
