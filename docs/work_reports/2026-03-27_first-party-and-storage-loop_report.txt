Party flow now supported
- Press `P` in the overworld to open the Party screen
- View the current party in slot order
- Move selection with `Up/Down`
- Press `Enter` to toggle a lightweight detail summary
- Press `R` or `Tab` from Party to inspect Storage
- Press `Esc` to return cleanly to the overworld

Storage/PC flow now supported
- Runtime storage now exists as a simple single-box creature list
- When capture succeeds and the party has room, the creature joins the party
- When capture succeeds and the party is full, the creature is sent to storage instead of being blocked
- Storage contents can be inspected from the Party screen through the new Storage screen
- Storage currently supports viewing only; no transfer/reorder/release actions were added in this task

Presentation/UI improvements
- Added dedicated Party and Storage screens with panel framing consistent with the current title/world/dialogue/battle style
- Added lightweight creature portrait treatments for party and storage lists
- Added a cleaner summary/detail panel for the selected creature
- Improved overworld HUD messaging so party/storage status is visible from the field
- Added a small PC-like environmental marker in the starter town to support the storage fantasy without introducing a full PC interaction system yet

Remaining issues
- Storage is currently a minimal single-box runtime list with no move/swap/release workflow yet.
