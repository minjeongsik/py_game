원격 수동 QA 세팅 안내

목적
- Codex가 게임 창을 직접 보고 입력하면서 새 게임부터 저장/Continue까지 수동 QA를 수행할 수 있도록 원격 접속 환경을 연다.

권장 방식
- 가장 간단한 방법은 AnyDesk 또는 RustDesk 사용
- Windows 원격 데스크톱은 네트워크/방화벽/계정 조건이 더 까다로워서 첫 시도용으로는 비권장

1. AnyDesk로 열기

설치
1. 브라우저에서 https://anydesk.com 접속
2. Windows용 AnyDesk 설치
3. 설치 후 AnyDesk 실행

설정
1. 메인 화면에서 Your Address 또는 이 장치의 주소 확인
2. 설정 > 보안 으로 이동
3. 무인 액세스 허용 또는 비슷한 항목 활성화
4. 접속 비밀번호 설정
5. 아래 권한이 있으면 같이 허용
- 키보드/마우스 제어 허용
- 화면 보기 허용
- 클립보드 허용은 선택

권장 추가 설정
- AnyDesk를 관리자 권한으로 설치 또는 실행
- 절전 모드 끄기
- 자동 화면 잠금 끄기
- 게임 실행 가능한 Windows 로그인 상태 유지

전달해야 할 정보
- AnyDesk 주소
- 접속 비밀번호
- 게임 폴더가 E:\min\job\ai\py_game 인지 여부

2. RustDesk로 열기

설치
1. 브라우저에서 https://rustdesk.com 접속
2. Windows용 RustDesk 설치
3. 설치 후 실행

설정
1. 메인 화면의 ID와 Password 확인
2. 설정 > 보안 에서 원격 제어 허용
3. 키보드/마우스 입력 허용

전달해야 할 정보
- RustDesk ID
- Password

3. Windows 원격 데스크톱으로 열기

설정
1. Windows 설정 > 시스템 > 원격 데스크톱
2. 원격 데스크톱 사용 켜기
3. 접속 허용 사용자 확인
4. PC 이름 확인
5. 방화벽에서 원격 데스크톱 허용 확인
6. 같은 네트워크 또는 VPN/포트포워딩 가능한 환경 확인

전달해야 할 정보
- PC 이름 또는 IP
- 사용자 계정명
- 비밀번호

주의
- 이 방식은 네트워크 조건 때문에 실패할 수 있어 AnyDesk 또는 RustDesk보다 우선순위가 낮음

세팅 완료 후 유지할 상태
- Windows 로그인 완료 상태 유지
- E:\min\job\ai\py_game 접근 가능
- 필요하면 터미널 하나 열어 두기
- 게임은 아직 실행하지 않아도 됨
- 절전/자동 잠금 해제

원격 접속 후 Codex가 수행할 수동 QA 범위
1. dotnet run --project PyGame.csproj 실행
2. 새 게임 시작
3. 집/도구점/치료소 진입 및 퇴장 확인
4. 해솔길 이동 확인
5. 야생전에서 PP/명중/경험치 확인
6. 레벨업 메시지 확인
7. 2마리 트레이너전 확인
8. 전투 중 가방 대상 선택 확인
9. 속삭임 숲과 바람개울 마을 이동 확인
10. 저장 후 종료
11. Continue 복원 확인
12. 한국어 UI 가독성 확인

보안 권장
- 임시 비밀번호로 열고 QA 종료 후 바로 비밀번호 변경
