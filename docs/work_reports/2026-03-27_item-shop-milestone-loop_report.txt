Summary

Added the first real item, shop, and badge-like milestone layer on top of the current prototype. The player can now open a Bag from the overworld, use healing items, buy supplies from a shopkeeper, spend and earn money, consume capture spheres during capture attempts, and claim a first local Route Mark milestone after clearing the route trainer challenge.

Item/inventory flow now supported

- Added item definitions for Potion and Capture Sphere.
- The Bag can now be opened from the overworld with B.
- Potion can be used on the current lead creature from the Bag.
- Battle now includes an ITEMS branch for Potion use.
- Capture now consumes a Capture Sphere and fails cleanly when none are available.

Shop/currency flow now supported

- Added a shopkeeper NPC in town that opens a small mart screen.
- The mart supports buying Potion and Capture Sphere one at a time.
- Money is tracked in runtime session state.
- Trainer victory now awards money, feeding the shop loop.
- Not having enough money is handled with direct feedback.

First milestone progression now supported

- Talking to the Guide still gives the first route challenge objective.
- After beating Scout Rin, talking to the Guide now grants a named local Route Mark-style milestone.
- The milestone awards one progression mark, bonus money, and a Potion.
- The milestone is tracked with progression flags and badge-count style state.

Presentation/refinement improvements

- Added Bag and Mart screens with matching retro panel framing.
- World HUD now shows money and mark count more clearly.
- Shopkeeper presence makes the town read more like a starter settlement instead of only a test area.
- Feedback text was tightened for buying, using items, earning money, and clearing the milestone.

Optimization-minded improvements

- Bag and battle item logic reuse existing inventory structures directly instead of introducing parallel runtime models.
- Shop inventory uses a static item id list rather than rebuilding display arrays each frame.
- Capture-sphere consumption was integrated into the existing capture path instead of splitting capture into a second system.
- World interactions continue to reuse the current NPC/state flow with only small targeted branches.

Remaining issues

- Bag use currently targets only the active lead creature and does not yet support per-party item targeting.
- Shop inventory is intentionally tiny and single-location only.
- The first milestone is a local route mark rather than a full gym/badge structure with separate case or ceremony flow.
