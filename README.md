# Aether Trail (MonoGame Prototype)

오리지널 2D 몬스터 수집 RPG를 목표로 하는 C# + MonoGame 프로젝트입니다.
현재는 **첫 번째 플레이 가능한 골격(Vertical Slice 0.1)** 으로,
타이틀 화면 -> 월드 진입 -> 이동/충돌/랜덤 인카운터 트리거 흐름을 제공합니다.

## 기술 스택
- C# (.NET 8)
- MonoGame DesktopGL
- JSON 기반 콘텐츠/세이브 데이터

## 실행 방법
1. .NET 8 SDK 설치
2. 저장소 루트에서 실행
   - `dotnet restore`
   - `dotnet build`
   - `dotnet run`

## 조작
- `Enter`: 타이틀 -> 월드 진입, 인카운터 오버레이 종료
- `방향키 (↑↓←→)`: 이동
- `WASD`: 이동 (방향키와 동일)
- `Esc`: 일시정지 메뉴 열기/닫기
- `F5`: 저장
- `F9`: 로드

## 현재 구현 범위
- 게임 상태 관리(상태 객체 구조)
  - Title / WorldExploration / PauseMenu / EncounterOverlay
- 월드 탐험
  - 탑다운 2D 이동 (방향키 + WASD)
  - 충돌 타일
  - 카메라 따라오기
  - JSON 맵 로딩
  - 안전 스폰 타일 보정(스폰이 막힌 타일이면 첫 번째 보행 가능 타일로 보정)
- 랜덤 인카운터 골격
  - 특수 지형 타일(맵의 `2`)에서 이동 중 확률 트리거
- 저장/로드 골격
  - 플레이어 위치 / 파티 / 인벤토리 구조
- 데이터 모델 골격
  - CreatureSpeciesDefinition / CreatureInstance / MoveDefinition / ItemDefinition / ZoneDefinition

## 디버그 오버레이
화면 하단에 다음 정보가 표시됩니다.
- 현재 게임 상태
- 현재 지역 이름
- 플레이어 월드 좌표 및 타일 좌표/타일 ID
- 입력 감지 여부, 최근 입력 방향 벡터, 실제 이동 적용 여부
- 윈도우 활성 상태(포커스)

## 로컬 테스트 체크리스트
1. 실행 후 타이틀 화면에서 `Enter`로 월드 진입
2. `↑↓←→` 입력 시 `POS` 값이 계속 변하는지 확인
3. `WASD` 입력도 동일하게 이동하는지 확인
4. 벽 타일(`ID:1`)에 부딪히면 이동이 막히는지 확인
5. 이동 시 카메라가 플레이어를 따라오는지 확인
6. 특수 지형(`ID:2`)에서 이동 중 인카운터 오버레이가 뜨는지 확인
7. 다른 창 클릭 후 돌아왔을 때 `WINDOW ACTIVE` 값이 `YES`로 복귀하는지 확인

## 폴더 구조
- `Core/`: 입력, 카메라, 상태 타입/매니저, 상태 컨텍스트, 상태 로직
- `World/`: 맵/로더, 플레이어 이동, 인카운터, 월드 렌더링
- `Creatures/`: 크리처 종/개체 데이터 모델
- `Data/`: 기술/아이템/지역/세이브 모델 및 저장 서비스(JSON)
- `UI/`: 화면 오버레이, 픽셀 폰트 기반 디버그 텍스트 렌더링
- `Content/Data/`: JSON 콘텐츠(테스트 맵)
